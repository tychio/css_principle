Scalable and Modular Architecture for CSS (SMACSS)
What is SMACSS?

It is a way to examine your design process.

It is an attempt to document a consistent approach to site development when using CSS.

SMACSS has two core goals

To increase the semantic value of a section of HTML and content.

To decrease the expectation of a specific HTML structure.

Base, Layout, Module, State and Theme

Base rules wouldn’t be applied to classes or IDs as they’re default CSS rules.

Layout rules are for the major components of a site.

Minor layout components are considered as modules and typically live inside the major layout components.

State rules apply to changes.

Theme rules are typically used least on a single project and aren’t considered one of the core categories.
